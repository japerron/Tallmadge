Include the requested attributes when the user requests them explicitly (breakdown, identify, list, top). Do not automatically include attributes from the filter in the output. Only include attributes in the output if they are requested explicitly.

Many questions will ask to filter on attributes in the customer and/or product dimensions. For each case, you should only use one column from the requested dimension(s) in the where clause, as follows: a) When deciding on which customer dimension column to use to filter data, filter only on the first column that contains the data, evaluating them in this order: first Customer Level 6, then 5, and so on down to 1. b) When deciding on which product dimension column to use to filter data, filter only on the first column that contains the data, evaluating them in this order: first 'Product Level 5 Segment', then Product Level 3, and so on down to 1.  When multiple products match the filter given, use a partial match criteria.
